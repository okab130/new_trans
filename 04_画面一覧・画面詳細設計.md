### 2.1 共通画面

#### CMN-001: ログイン画面

**画面レイアウト**
```
+----------------------------------+
|   トラック配送管理システム        |
|                                  |
|   [ロゴ]                         |
|                                  |
|   ログインID                      |
|   [________________]             |
|                                  |
|   パスワード                      |
|   [________________]             |
|                                  |
|   [ ] ログイン状態を保持する      |
|                                  |
|   [    ログイン    ]             |
|                                  |
|   パスワードを忘れた方はこちら    |
+----------------------------------+
```

**入力項目**
| 項目名 | 必須 | 形式 | バリデーション |
|--------|------|------|----------------|
| ログインID | ○ | テキスト | 半角英数字、最大50文字 |
| パスワード | ○ | パスワード | 最小8文字 |
| ログイン状態保持 | - | チェックボックス | - |

**処理フロー**
1. ログインID・パスワード入力
2. [ログイン]ボタン押下
3. 認証処理（JWT発行）
4. ロールに応じたダッシュボードへ遷移

**エラー処理**
- ログインID未入力: "ログインIDを入力してください"
- パスワード未入力: "パスワードを入力してください"
- 認証失敗: "ログインIDまたはパスワードが正しくありません"
- アカウントロック: "アカウントがロックされています。管理者に連絡してください"

```
+----------------------------------------------------------+
```


**入力項目**
| 項目名 | 必須 | 形式 | バリデーション |
|--------|------|------|----------------|
| 依頼日 | ○ | 日付 | 本日以前 |
| 希望配送日 | ○ | 日付 | 依頼日以降 |
| 配送時間帯 | - | ラジオボタン | - |
| 指定時刻FROM/TO | △ | 時刻 | 時間指定選択時必須、FROM < TO |
| 顧客 | ○ | 選択 | マスタから選択 |
| 配送先 | ○ | 選択 | 選択顧客に紐づく配送先 |
| 品目明細 | ○ | 表 | 1件以上必須 |
| 配送種別 | ○ | ラジオボタン | デフォルト:通常 |
| 温度帯 | - | ラジオボタン | - |
| 優先度 | ○ | セレクト | HIGH/NORMAL/LOW |
| 特記事項 | - | テキストエリア | 最大500文字 |

**処理フロー**
1. 各項目入力
2. [登録]ボタン押下
3. バリデーション実行
4. 確認ダイアログ表示
5. 登録処理実行
6. 完了メッセージ表示
7. 配送依頼一覧へ遷移

**バリデーション**
- 必須項目チェック
- 日付の前後関係チェック
- 品目明細が1件以上あること
- 重量・容積の妥当性チェック（警告レベル）

## 3. UI/UX設計ガイドライン

### 3.1 デザイン原則
- **シンプル**: 必要最小限の情報表示
- **直感的**: 説明不要で操作可能
- **一貫性**: 全画面で統一されたデザイン
- **アクセシブル**: 高齢者・色覚多様性に配慮

### 3.2 カラースキーム
```
プライマリカラー: #2196F3 (青)
セカンダリカラー: #FF9800 (オレンジ)
成功: #4CAF50 (緑)
警告: #FFC107 (黄)
エラー: #F44336 (赤)
背景: #FAFAFA (薄グレー)
テキスト: #333333 (濃グレー)
```

### 3.3 フォント
- 日本語: Noto Sans JP
- 英数字: Roboto
- サイズ: 14px（本文）、16px（見出し）、12px（補足）

### 3.4 レスポンシブブレークポイント
- モバイル: 〜768px
- タブレット: 769px〜1024px
- デスクトップ: 1025px〜

### 3.5 操作フィードバック
- ボタン押下: リップル効果
- 処理中: ローディングスピナー
- 成功: トーストメッセージ（緑）
- エラー: トーストメッセージ（赤）
- 警告: 確認ダイアログ

### 3.6 モバイルUI特有の配慮
- タップ領域: 最小44x44px
- スワイプジェスチャー: リスト削除、更新
- プルリフレッシュ: データ再読込
- ボトムナビゲーション: 主要機能へのアクセス
- フローティングアクションボタン: 新規登録等
