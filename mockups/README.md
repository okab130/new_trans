# トラック配送計画・配送実績追跡システム - 画面モックアップ

## モックアップ一覧

このディレクトリには、トラック配送計画・配送実績追跡システムの主要画面のHTMLモックアップが含まれています。
各モックアップはスタンドアロンで動作し、ブラウザで直接開いて確認できます。

### Web版（PC・タブレット向け）

#### 1. ログイン画面
**ファイル**: `01_login.html`
- システムへのログイン
- ログインID・パスワード入力
- ログイン状態保持オプション

#### 2. ダッシュボード（配車担当者向け）
**ファイル**: `02_dashboard.html`
- 本日のサマリー表示（配送予定、完了、配送中、問題）
- クイックアクション
- 配送進捗状況
- 定時配送率
- 直近のアラート表示

#### 3. 配送依頼一覧
**ファイル**: `03_delivery_order_list.html`
- 配送依頼の検索・フィルタリング
- 一覧表示（ページネーション対応）
- ステータス別表示
- 詳細・編集・削除操作

#### 4. 配車計画作成
**ファイル**: `04_route_planning.html`
- 未割当配送依頼の表示
- 配車済ルートの表示
- ドラッグ&ドロップによる割当（UI表現）
- 車両・ドライバー情報
- 積載容量の可視化
- ルート最適化実行

#### 5. 配送進捗モニター
**ファイル**: `05_tracking_monitor.html`
- リアルタイム地図表示
- 車両位置の表示
- 配送先マーカー（完了/配送中/未到着）
- ルート別ステータス一覧
- 遅延アラート表示

### モバイル版（スマートフォン向け）

#### 6. 配送計画確認（ドライバー用）
**ファイル**: `06_mobile_delivery_plan.html`
- 当日の配送計画表示
- 配送進捗の可視化
- 次の配送先情報
- 配送先リスト
- ナビゲーション連携ボタン

#### 7. 配送実績入力（ドライバー用）
**ファイル**: `07_mobile_delivery_input.html`
- 配送結果の選択（完了/一部配送/配送不可）
- 配送時刻の記録
- 受領者情報の入力
- 配送証跡（写真・サイン）取得ボタン
- 備考入力

#### 8. 電子サイン取得（ドライバー用）
**ファイル**: `08_mobile_signature.html`
- タッチパネルでのサイン入力
- JavaScript実装済み（実際に描画可能）
- サインのクリア機能
- サイン画像の保存

### 分析・管理画面

#### 9. KPIダッシュボード
**ファイル**: `09_kpi_dashboard.html`
- 主要KPIの表示（定時配送率、配送効率、問題発生率）
- 目標達成率の可視化
- グラフ表示（棒グラフ、円グラフ）
- タブ切替（配送実績/車両稼働/ドライバー/コスト）
- レポート出力・データエクスポート

## モックアップの使い方

### ローカルでの確認方法

1. 任意のモックアップファイルをブラウザで開く
   ```
   例: 01_login.html をダブルクリック
   ```

2. または、簡易HTTPサーバーで起動
   ```bash
   # Python 3の場合
   python -m http.server 8000
   
   # Node.jsの場合
   npx http-server
   ```
   ブラウザで `http://localhost:8000/` にアクセス

### モックアップの特徴

- **レスポンシブデザイン**: モバイル版は最大幅414px（iPhone Pro Max相当）
- **スタンドアロン**: 各ファイルは独立して動作
- **CSS内蔵**: 外部CSSファイル不要
- **インタラクティブ**: 一部JavaScriptで動作を実装（電子サイン等）

## 実装時の注意点

### Web版
1. **ヘッダーの固定**: `position: sticky` または `position: fixed` を使用
2. **テーブルのレスポンシブ対応**: 横スクロール実装を検討
3. **地図API連携**: Google Maps API または Leaflet の組み込み
4. **リアルタイム更新**: WebSocket または Server-Sent Events の実装

### モバイル版
1. **タップ領域**: 最小44x44pxを確保
2. **スワイプジェスチャー**: リスト削除・更新機能
3. **プルリフレッシュ**: データ再読込機能
4. **オフライン対応**: Service Worker でのキャッシュ戦略
5. **位置情報取得**: Geolocation API の利用
